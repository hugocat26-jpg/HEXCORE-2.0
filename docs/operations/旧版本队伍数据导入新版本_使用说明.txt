旧版本队伍数据导入新版本使用说明

一、推荐方式：导入完整状态备份

1. 打开旧版本 HEXCORE。
2. 点击左侧菜单“系统设置”。
3. 点击“导出状态备份”。
4. 保存导出的 .json 状态备份文件。
5. 打开新版本 HEXCORE。
6. 点击左侧菜单“系统设置”。
7. 点击“导入状态备份”。
8. 选择刚才从旧版本导出的 .json 文件。
9. 导入完成后，进入“队伍管理”检查队长、队员、海克斯是否正确。
10. 再进入“系统设置”，运行系统检查，确认没有重复归属、跨阵营、缺员等异常。

二、只导入选手名单

如果只想导入选手名单，不想覆盖当前比赛状态，请不要导入状态备份。

操作步骤：

1. 打开新版本 HEXCORE。
2. 点击左侧菜单“选手库”。
3. 点击“批量导入”。
4. 选择 JSON 或 CSV 选手名单文件。
5. 在导入预览中检查新增、跳过、重复 ID、缺字段和非法评分等提示。
6. 确认无误后执行导入。
7. 导入后进入“队伍管理”，重新设置队长和队员。

三、导入前注意事项

1. 导入状态备份会覆盖当前新版本页面里的本地状态。
2. 导入前，建议先在新版本点击一次“导出状态备份”，保留当前状态，方便回退。
3. 不建议直接修改浏览器 localStorage，也不建议直接覆盖项目文件。
4. 如果旧版本规则差异较大，导入后请重点检查：
   - 队长是否正确。
   - 队员是否归属正确。
   - 队伍是否满员。
   - 海克斯是否还可用。
   - 金币、购买权和刷新状态是否符合现场进度。
   - 赛程是否需要重新生成。

四、常见情况

1. 旧版本导出的完整状态备份：
   使用“系统设置 -> 导入状态备份”。

2. 只有选手名单，没有比赛状态：
   使用“选手库 -> 批量导入”。

3. 导入后队伍异常：
   先运行系统检查，再根据提示处理重复归属、跨阵营、缺员或无队长问题。

4. 导入后赛程不符合当前新赛制：
   建议清空赛程后重新生成。班德尔保卫战请使用“赛程 -> 生成班德尔保卫战”。

一句话说明：

有旧版 .json 状态备份，就走“系统设置 -> 导入状态备份”；只有名单文件，就走“选手库 -> 批量导入”。
