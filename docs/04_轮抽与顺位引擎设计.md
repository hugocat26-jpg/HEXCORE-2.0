# 轮抽与顺位引擎设计

## 1. 基础规则

- 每轮对应一个基础卡池。
- 第1轮：侏儒马。
- 第2轮：中等马。
- 第3轮：上等马。
- 第4轮：猛犸。
- 每位队长每轮默认获得一次抽卡和选人机会。
- 队伍满员后自动跳过后续抽卡和选人。
- 不存在金币、购买、破产。

## 2. 基础蛇形顺位

```text
第1轮：A B C D
第2轮：D C B A
第3轮：A B C D
第4轮：D C B A
```

## 3. 顺位修改优先级

顺位类效果必须使用优先级，不允许按执行顺序碰运气。

建议优先级：

```text
1000 裁判强制修正
900  背水一战第4轮第一顺位
800  启元本轮优先队列
700  潘多拉固定第3顺位
600  恶魔契约 1-3轮第一 / 第4轮最后
500  开饭啦临时优先选人
400  顺位互换，只影响基础顺位
100  普通蛇形顺位
```

## 4. TurnOrderModifier

```ts
export type TurnOrderModifier = {
  id: string;
  source: 'hexcore' | 'rule' | 'referee';
  sourceId: string;
  priority: number;
  captainId?: string;
  round?: number;
  operation:
    | 'move_first'
    | 'move_last'
    | 'fixed_position'
    | 'swap_base_order'
    | 'move_back'
    | 'skip'
    | 'insert_temporary_turn';
  payload?: Record<string, unknown>;
};
```

## 5. 顺位计算流程

```text
生成基础蛇形顺位
  ↓
应用基础顺位互换
  ↓
移除队伍已满队长
  ↓
应用跳过类效果
  ↓
应用固定顺位类效果
  ↓
应用优先队列类效果
  ↓
插入临时补偿回合
  ↓
输出最终顺位和解释
```

## 6. 顺位解释

每个队长的最终位置应可解释：

```ts
type TurnOrderExplanation = {
  captainId: string;
  basePosition: number;
  finalPosition: number;
  reasons: string[];
};
```

示例：

```text
c7：
- 基础顺位第5
- 恶魔契约：第1-3轮移动到第1
- 启元使用者优先级高于恶魔契约，最终位置第2
```

## 7. 抽卡流程

```text
turn_start
  ↓
检查队伍是否满员
  ↓
检查是否跳过
  ↓
检查是否自动分配
  ↓
计算有效卡池
  ↓
计算抽卡候选
  ↓
应用概率和展示效果
  ↓
生成 DrawState
  ↓
裁判选择卡牌
  ↓
Assignment Engine 入队
  ↓
turn_end
```

## 8. 动态临时回合

盲盒等效果会产生补偿回合。

```ts
export type TemporaryTurn = {
  id: string;
  insertedAfterCaptainId: string;
  captainId: string;
  round: number;
  tier: number;
  reason: string;
};
```

补偿回合必须进入事件日志，不能只在内存里临时插入。
