# 开发门禁

更新时间：2026-05-30

本文记录所有开发任务完成前必须执行的固定门禁。当前待开发事项见 [当前待开发计划.md](当前待开发计划.md)。

## 通用门禁

| 门禁 | 验收标准 |
| --- | --- |
| 回归测试 | `npm test` 通过，无新增异常用例。 |
| 语法检查 | 本次涉及的 JavaScript 文件执行 `node --check`。 |
| Diff 检查 | 本次改动文件执行 `git diff --check -- <files>`，不得有冲突标记、异常 EOF 或 trailing whitespace。 |
| 状态保护 | 新功能不得破坏本地状态保存、导入、导出、撤销和多人端恢复。 |
| 阵营保护 | 购买、补位、海克斯、转移、自动分配都不能造成跨阵营入队，除非产品规则显式改变。 |
| 队长保护 | 队长锁定选手不可被商店、补位、购买或海克斯作为队员获得。 |
| 容量保护 | 任意动作后队伍人数不得超过容量。 |
| 日志审计 | 影响选手、金币、顺位、卡池、海克斯状态、计时和权限的动作必须写事件日志。 |
| UI 可解释 | 按钮不可用时必须有明确原因，不能静默无反馈。 |

## 前端与浏览器门禁

- 涉及前端、交互、浏览器验证、UI 状态、布局和视觉改造时，使用 Build Web Apps 相关技能。
- 针对已渲染页面的修复、调试和交互验证，优先使用 `build-web-apps:frontend-testing-debugging` 和 Browser 插件。
- 页面验证至少覆盖：页面非空、身份正确、无错误覆盖层、console 无 error/warn、关键交互可操作。
- 手机或局域网体验相关改动，应额外检查窄视口或真实局域网访问说明。

## 安全门禁

涉及以下边界时必须做差异范围安全审查：

- 鉴权、session、streamToken、房间码、加入码。
- command payload、公开投影、SSE、WebSocket 或实时同步。
- 导入、导出、备份、恢复、数据库、部署脚本。
- 状态归一化、删除、重置、回滚、计时超时自动推进。
- 任何用户可控输入进入 HTML、日志、URL、命令或文件路径。

安全审查发现可成立问题时，必须先修复并重新验证；不得在存在未处理安全发现时标记计划完成或推送 GitHub。

## 推送门禁

- 改动较大且验证通过后，先更新版本号，再提交并推送 GitHub。
- `CHANGELOG.md` 必须同步用户可见变化、安全边界变化和部署方式变化。
- 小文档整理可不立即推送，但不得让权威文档与实际流程相互矛盾。

## 任务钩子

默认任务钩子读取当前计划：

```powershell
npm run post-task -- --status=complete
```

也可显式指定当前计划文档：

```powershell
npm run post-task -- --doc=docs/planning/当前待开发计划.md --status=complete
```

旧兼容入口仍可显式指定：

```powershell
npm run post-task -- --doc=docs/06_开发计划.md --status=complete
```
