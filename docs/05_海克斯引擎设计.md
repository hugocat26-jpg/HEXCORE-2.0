# 海克斯引擎设计

## 1. 设计目标

海克斯引擎负责把海克斯定义转化为可执行规则。

目标：

- 尽量自动执行。
- 支持卡池、顺位、概率、展示、自动分配、转队等复杂效果。
- 支持每轮询问。
- 支持裁判代执行。
- 支持未来队长自行操作。

## 2. 规则原语

所有海克斯应拆为以下规则原语：

```text
PoolModifier        修改卡池
TurnOrderModifier   修改顺位
DrawModifier        修改抽卡数量、范围或抽卡方式
PickModifier        修改选人方式
AssignmentEffect    自动入队、转队、补偿
VisibilityModifier  暗牌、毛玻璃、延迟揭示
PromptEffect        每轮询问、选择目标
DisableEffect       禁用其他类型海克斯
AnnouncementEffect  公告
```

## 3. HexcoreDefinition

```ts
export type HexcoreDefinition = {
  id: number;
  name: string;
  rarity: 'silver' | 'gold' | 'prismatic';
  trigger:
    | 'on_acquire'
    | 'round_start'
    | 'round_start_prompt'
    | 'before_turn'
    | 'before_draw'
    | 'after_draw'
    | 'before_pick'
    | 'after_pick'
    | 'manual';
  rounds?: number[];
  maxUses: number;
  maxUsesPerRound?: number;
  effects: HexcoreEffect[];
  tags?: string[];
};
```

## 5. 致盲和暗信息

当前阶段由裁判代执行，裁判可以看见真实卡牌。

但引擎仍需支持未来队长端：

- 被致盲队长视图不包含真实 `playerId`、姓名、分数。
- 裁判视图包含真实卡牌。
- 选中后才公开揭示。

这要求抽卡结果区分“内部真实状态”和“角色可见视图”。

## 6. PromptEffect

每轮询问类海克斯包括：

- 启元。
- 摄影艺术家。
- 致盲吹箭。
- 锁定契约。
- 雪定饿的喵。

当前阶段 Prompt 由裁判代替队长回答：

```text
系统展示询问
  ↓
裁判获得线下答复
  ↓
裁判点击 是 / 否 / 选择目标
  ↓
系统生成事件
```

## 7. 冲突处理

海克斯冲突必须显式处理。

示例：

- 潘多拉禁用开饭啦、启元等自主选人类效果。
- 背水一战第4轮第一顺位优先级高于启元。
- 启元优先级高于恶魔契约。
- 顺位互换只能作用于基础顺位，不能交换固定顺位效果。

冲突处理结果应写入顺位解释或事件日志。
