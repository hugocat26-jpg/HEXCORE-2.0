# 海克斯引擎设计

更新时间：2026-05-25

本文档记录当前金币商店模式的海克斯引擎口径。裁判端维护版仍以阵营锁定 10 队为稳定基线；多人端新增可配置队伍和无阵营模式。旧版无金币轮抽海克斯、旧补偿回合、跨阵营转队和旧顺位优先级不能作为当前实现依据。

## 1. 设计目标

海克斯引擎负责把 `src/core/sample-data.js` 中的海克斯定义转化为可执行规则，并保证所有结果符合金币商店模式的阵营、队长、容量、购买权和日志审计约束。

目标：

- 支持裁判代执行的手动海克斯。
- 支持获得、轮初、商店、刷新、购买后、轮末等被动触发。
- 支持目标选择、失败原因、执行队列和响应窗口。
- 支持顺位、商店、金币、暗信息、入队和阵容置换类效果。
- 双阵营模式下，所有入队结果必须经过同阵营、非队长、容量和禁用状态校验。
- 无阵营模式下，入队结果改走公共候选池，不显示同阵营、本阵营、异阵营等误导文案。
- 冲突处理结果必须写入事件日志或顺位解释。

## 2. 当前规则原语

当前海克斯可拆为以下规则原语：

```text
ShopModifier         修改商店张数、候选、保留、替换或显示身份
EconomyModifier      修改金币、折扣、返还、免费刷新
OrderModifier        修改轮初顺位、跳过、响应窗口
VisibilityModifier   隐藏或打乱显示信息，购买后揭示真实选手
RosterModifier       直接入队、随机补强、整队置换
RuntimeEffect        跨开店、跨购买、跨轮次保存的运行时效果
ResponseWindow       购买后的限时响应入口
DisableGuard         海克斯启禁和旧海克斯白名单过滤
```

## 3. HexcoreDefinition

当前定义结构以 `sample-data.js` 为准：

```ts
export type HexcoreDefinition = {
  id: string;
  name: string;
  type: 'cyan' | 'amber' | 'violet';
  category:
    | 'shop_control'
    | 'economy'
    | 'disruption'
    | 'roster_replace'
    | 'order_response';
  tags: string[];
  mode: 'manual' | 'passive';
  uses: number;
  maxUsesPerRound?: number;
  needsTarget?: 'captain' | 'player' | 'shopCard';
  desc: string;
};
```

当前阵营锁定金币模式只执行 `hexcore-engine.js` 中 `CAMP_HEXCORE_IDS` 白名单内海克斯。旧海克斯和白名单外配置不能通过 UI 或动作层执行。

## 4. 当前海克斯自动化覆盖

| 海克斯 | 当前自动化方式 |
| --- | --- |
| 阵营侦察 | 开店前登记，下次商店额外展示 1 张同阵营卡 |
| 压价券 | 当前商店购买前使用，本次购买费用 -1，最低 1 |
| 保留席位 | 当前商店选择 1 张卡，下次刷新保留 |
| 加急调货 | 当前商店选择 1 张卡，替换为同阵营同费用且当前商店外可选卡 |
| 阵营封锁 | 目标下一次商店少 1 张，最低 3 张 |
| 抬价干扰 | 目标下一次购买费用 +1，UI 显示醒目标记 |
| 稳健补强 | 商店打开前按轮次最低费用门槛随机获得 1 名同阵营选手并消耗购买权 |
| 捐赠 | 获得时初始资金 +2 |
| 赞助回流 | 每局最多 2 次，购买实付不低于 3 时返还 1 金币 |
| 海浪，我没吃饭 | 每轮随机 1 名持有者触发，金币清零、跳过本轮、免疫目标型海克斯，并判定后续购买 |
| 背水一战 | 已有 4 名队员后弹窗确认，从本阵营合法候选中随机置换 4 人 |
| 开饭啦 | 第 3 轮开始时 +3 金币 |
| 吸血习性 | 从金币最高且金币大于 0 的三名其他队长处各获得 1 金币 |
| 巨人杀手 | 首次购买 4 费和 5 费时各优惠 1 金币 |
| 摄影艺术家 | 每轮额外 1 次不可累计免费刷新 |
| 贤者的博爱 | 每个己方选人阶段获得当前轮数金币和 1 次可累计刷新 |
| 神秘贤者·启元 | 获得时 +2；每轮开始自动提到第一顺位 |
| 神秘贤者·盲盒 | 商店打开前支付 3 金币，从同阵营 2-5 费可选池随机入队 1 人 |
| 质变：黄金阶 | 商店打开前从同阵营 4 费池随机入队 1 人 |
| 质变：棱彩阶 | 商店打开前从同阵营 5 费池随机入队 1 人 |
| 知识来源于分解 | 每轮 +1 解构，满 3 层后可自选高费；金币不足可分解 2/3 费队员抵扣 |
| 和我困在一起 | 指定同阵营可选选手，若到下一轮仍可选则直接入队并消耗购买权 |
| 骤雨 血雾 清风 | 按顺位影响最多 3 名仍有购买权且未满员的非使用者队长，刷新不清除 |
| 雪定饿的喵 | 目标下一次商店显示身份被打乱，费用不参与打乱，购买后揭示真实选手 |
| 大炮已充能 | 轮初弹窗二选一：自己前移 1 位或目标后移 1 位 |
| 神兵天降 | 其它队长购买后 10 秒内夺取同阵营购买，成功入队后发动者跳过下一轮 |

`舞会女王` 仍保留数据和部分商店逻辑，但不在当前阵营锁定金币模式执行白名单中，不应作为当前可执行海克斯描述。

## 5. 多人端无阵营改写

多人端 `settings.campMode = 'no_camp'` 时，不整体禁用海克斯，而是采用公共池改写：

- `阵营侦察`、`保留席位`、`加急调货`、`稳健补强`、`舞会女王`、`神秘盲盒`、`黄金阶`、`棱彩阶`、`知识来源于分解`、`和我困在一起`、`背水一战` 的候选池从同阵营/本阵营改为公共可选池。
- `海浪，我没吃饭` 命中任意其他队长购买后直接夺取，原购买队长返还金币、购买权和补偿；取消异阵营退回后轮末同阵营补偿分支。
- `神兵天降` 在响应窗口内可夺取任意其他队长刚购买的选手；仍限制每局 1 次，成功后发动者跳过下一轮。
- 服务端规则、SSE 投影、事件日志和 UI 详情必须使用同一套改写结果，不能只在前端替换文案。

## 6. 暗信息和显示身份

当前裁判端能看到真实状态，但 UI 仍必须区分真实状态和显示状态：

- 真实入队以 `playerId` 为准。
- `displayPlayerId` 只用于受扰乱商店的显示身份。
- `雪定饿的喵` 只打乱本次商店内显示身份，不打乱费用。
- `骤雨 血雾 清风` 只影响商店可见性，不改变真实入队校验。
- 购买后需要在事件日志中记录显示身份、真实身份和实际扣费。

## 7. 目标选择

目标型海克斯必须通过统一目标校验：

- 双阵营模式下，选手目标默认只能选择同阵营、非队长、未入队、未禁用的合法候选。
- 无阵营模式下，选手目标默认从公共候选池选择。
- 队长目标必须仍有生效窗口，并满足队伍容量和购买权约束。
- `海浪，我没吃饭` 持有者不作为血雾和雷霆一击目标。
- 本轮已触发海浪的队长免疫其它目标型海克斯。
- `神秘贤者·启元` 持有者或本轮受启元保护/提位的队长不能成为雷霆一击目标。

## 8. 冲突处理

海克斯冲突必须显式处理。

当前重点规则：

- 金币模式下 `背水一战` 不再参与顺位优先级，只做本阵营整队置换。
- `背水一战` 不能抽取持有 `海浪，我没吃饭` 队伍当前拥有的队员。
- `大炮已充能` 在 `神秘贤者·启元` 关闭提示后才出现；大炮不能越过启元第一顺位。
- `雷霆一击` 目标不能是自己、当前最后顺位、启元持有/保护队长或海浪持有队长。
- `雪定饿的喵` 与 `加急调货` 叠加时，调货后必须重新映射显示身份，显示身份只允许来自当前商店真实卡。
- 双阵营模式下 `神兵天降` 只能夺取同阵营购买；无阵营模式下可夺取任意其他队长刚购买选手。
- `骤雨 血雾 清风` 只影响仍有购买权的队长；没有购买权则跳过，购买权恢复则继续响应。

## 9. 状态记录

海克斯执行必须写入以下状态之一：

- 海克斯自身状态：`used`、`lastUsedRound` 或被动计数。
- `draft.runtimeEffects`：跨商店、跨轮次或响应窗口效果。
- 事件日志：成功、失败、跳过、免疫、补偿和轮末奖励。
- UI 状态：轮初弹窗、详情弹窗、入队揭示、神兵响应窗口等临时展示状态。

## 10. 金币模式：大炮已充能

`大炮已充能` 是轮初顺位转换技，只能在每轮商店打开前处理，不进入选人阶段的海克斯执行队列。

执行顺序：

1. 轮次开始时先结算 `神秘贤者·启元`。
2. 关闭启元提示后，再按当前顺位询问持有 `大炮已充能` 且本轮未处理的队长。
3. 弹窗提供 `加速之门`、`雷霆一击`、`本轮不使用` 三个入口。
4. 弹窗未处理前禁止生成商店。

加速之门：

- 只能对持有者自己使用。
- 点击后进入预览页，显示当前顺位和确认后顺位。
- 确认后自己前移 1 位。
- 不能越过 `神秘贤者·启元` 形成的第一顺位保护。

雷霆一击：

- 只能在轮初弹窗内使用。
- 需要选择 1 名其它队长作为目标。
- 目标不能是自己、不能是当前最后顺位、不能是持有或本轮受 `神秘贤者·启元` 保护/提位的队长，也不能是持有 `海浪，我没吃饭` 的队长。
- 确认后目标本轮顺位后移 1 位。

确认加速、确认雷霆或跳过后，均标记该海克斯本轮已处理。
