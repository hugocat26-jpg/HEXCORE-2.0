# UI 设计规范

## 1. 当前阶段界面目标

当前阶段先实现裁判工作台 UI。所有操作由裁判执行，页面不提供队长登录入口。

主界面需要支持：

- 查看当前轮次、卡池、当前队长。
- 查看当前顺位和顺位变化原因。
- 裁判代执行抽卡、选卡、跳过、下一位、暂停、撤销。
- 查看当前队长可用海克斯并代为使用。
- 查看事件日志。
- 查看 5-20 支队伍的动态阵容概览。

## 2. 主屏布局

采用固定应用壳：

```text
左侧导航 220px
顶部状态栏 64px
中间主工作区
右侧事件日志 320px
底部队伍栏 120px
```

中间主工作区分为：

- 顶部顺位条。
- 左侧抽卡区。
- 右侧当前队长海克斯与规则说明。
- 底部裁判操作按钮。

## 3. 视觉方向

视觉风格为冷静、紧凑、专业的选人抽卡控制台。

色彩：

- 背景：深炭黑。
- 主边框：冷灰蓝。
- 主要强调：青色。
- 警告和顺位强调：琥珀色。
- 成功状态：绿色。
- 危险状态：红色。

避免：

- 紫色主导渐变。
- 营销页 hero。
- 大面积装饰背景。
- 过度圆角卡片。
- 嵌套卡片。
- 说明性教程文案。

## 4. 组件规范

### 导航

- 左侧竖向导航。
- 当前页面高亮青色背景。
- 每个导航项包含小图标和中文标签。

### 顶部状态栏

必须显示：

- HEXCORE 2.0。
- 裁判代执行。
- 当前阶段。
- 当前队长。
- 选人流程状态。
- 当前时间。
- 刷新按钮。

### 选手卡

每张卡显示：

- 序号。
- 分路或标签。
- 选手名。
- 游戏 ID。
- 评分。
- 近 5 场统计。
- 擅长英雄缩略槽。
- 选中状态。

### 海克斯面板

每个海克斯行显示：

- 图标。
- 名称。
- 效果摘要。
- 可用次数。
- 使用按钮或已使用状态。

### 事件日志

事件按时间倒序展示。

事件类型：

- 海克斯询问。
- 抽卡完成。
- 选手入队。
- 顺位变更。
- 裁判操作。

## 5. 响应式要求

桌面优先。

小屏幕处理：

- 左侧导航收缩为顶部横向。
- 右侧日志移动到底部。
- 选手卡从 3 列变为 1 列。
- 队伍阵容栏允许横向滚动。

## 6. 第一版交互范围

第一版 UI 原型只做本地演示交互：

- 切换选中卡。
- 点击抽卡刷新日志。
- 点击选择此卡更新队伍栏和日志。
- 点击海克斯按钮更新状态和日志。
- 点击下一位切换当前队长。

不实现：

- 真实抽卡概率。
- 完整海克斯引擎。
- 旧数据导入。
- 队长登录。
- 服务端同步。
