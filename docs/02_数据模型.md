# 数据模型

## 1. 设计原则

- 当前权威模式为 `economyMode: 'gold_shop'`。
- 队伍固定 10 队，每队总人数包含队长。
- `captains[].playerId` 是队长锁定选手；队长锁定选手不作为普通队员入队。
- `captains[].team` 只保存普通队员 `playerId`，不重复保存队长。
- 选手必须带 `camp`，入队、商店、补位和海克斯目标均以同阵营为硬约束。
- 费用档位使用 `tier: 1-5`；`tier: 0` 仅用于队长锁定展示。
- 所有关键操作写入事件日志，状态必须可序列化，便于本地保存、导入导出和后续迁移。

## 2. 基础类型

```ts
export type Camp = 'local' | 'outsider';
export type Tier = 0 | 1 | 2 | 3 | 4 | 5;
export type PlayerStatus = 'available' | 'drafted' | 'captain' | 'disabled';

export type DraftPhase =
  | 'setup'
  | 'hexcore_draw'
  | 'drafting'
  | 'completed'
  | 'paused';
```

阵营显示：

| 值 | 显示 |
| --- | --- |
| `local` | 本地人 |
| `outsider` | 外地人 |

## 3. Player

```ts
export type Player = {
  id: string;
  name: string;
  gameId: string;
  position?: string;
  heroes?: string[];
  slogan?: string;
  score: number;
  resultScore?: number;
  seasonResults?: {
    s1: string;
    s2: string;
    s3: string;
    s4: string;
    s5: string;
    s6: string;
  };
  fmvpSeasons?: string[];
  tier: Tier;
  tierSource: 'resultScore' | 'seasonResults' | 'score';
  camp: Camp;
  status: PlayerStatus;
  teamId?: string;
};
```

费用池规则：

- 队长锁定选手显示为 `0` 费，不进入普通商店池。
- 有官方成绩的选手按阵营内成绩排序后分入 `1-5` 费。
- 无官方成绩的选手按 `score` 直接映射到 `1-5` 费。
- 超过组队需求的选手保持 `available`，可作为商店候选或后续补位来源。

## 4. Captain

```ts
export type Captain = {
  id: string;
  name: string;
  playerId?: string;
  team: string[];
  gold: number;
  freeRefreshes: number;
  purchasedThisRound: boolean;
  skippedThisRound: boolean;
  status: 'active' | 'completed' | 'skipped';
};
```

队伍容量由 `settings.playersPerTeam` 决定。当前默认 `playersPerTeam = 5` 且包含队长，因此普通队员容量通常为 4。

## 7. ShopDraw

```ts
export type ShopDraw = {
  id: string;
  captainId: string;
  round: number;
  pickMode: 'shop';
  generatedBy: 'free_shop' | 'paid_refresh' | 'hexcore';
  slots: ShopSlot[];
};

export type ShopSlot = {
  slotId: string;
  playerId: string;
  displayPlayerId?: string;
  tier: 1 | 2 | 3 | 4 | 5;
  price: number;
  purchased: boolean;
  locked?: boolean;
  modifiers?: string[];
};
```

`displayPlayerId` 用于“雪定饿的喵”等显示身份打乱效果。购买结算以真实 `playerId` 为准，日志同时记录显示卡和真实卡。

## 8. Hexcore

```ts
export type HexcoreDefinition = {
  id: string;
  name: string;
  type: 'cyan' | 'amber' | 'violet';
  category:
    | 'shop_control'
    | 'economy'
    | 'disruption'
    | 'roster_replace'
    | 'order_response';
  tags: string[];
  mode: 'manual' | 'passive';
  uses: number;
  maxUsesPerRound?: number;
  needsTarget?: 'captain' | 'player' | 'shopCard';
  desc: string;
  detail?: string | string[];
  tips?: string[];
  notes?: string[];
};

export type HexcoreAssignments = Record<string, string[]>;
```

当前金币模式海克斯以 `src/core/sample-data.js` 为准。旧海克斯不进入默认池，也不能在金币模式入口选择。

## 9. RuntimeEffect

```ts
export type RuntimeEffect = {
  id: string;
  source: 'hexcore' | 'rule' | 'referee';
  sourceId: string;
  ownerCaptainId?: string;
  targetCaptainId?: string;
  type: string;
  priority: number;
  round?: number;
  expiresAt:
    | { type: 'round_end'; round: number }
    | { type: 'turn_end'; captainId: string; round: number }
    | { type: 'shop_open' }
    | { type: 'purchase_window'; seconds: number }
    | { type: 'session_end' };
  payload: Record<string, unknown>;
};
```

常见运行时效果包括商店加卡、商店减卡、抬价、显示身份打乱、轮初顺位变化、购买后响应和延迟入队。

## 11. TournamentState

```ts
export type TournamentState = {
  status: 'empty' | 'ready' | 'running' | 'completed';
  championId: string;
  rounds: TournamentRound[];
};

export type TournamentMatch = {
  id: string;
  sideAId: string;
  sideBId: string;
  scoreA: number | '';
  scoreB: number | '';
  winnerId: string;
  status: 'pending' | 'bye' | 'completed';
  byeConfirmed?: boolean;
};
```

赛程模块只记录选人完成后的阵营对抗框架和比分推进，不改变选人流程、队伍成员或金币状态。
