# 数据模型

## 1. 设计原则

- 不使用金币模型。
- 选手档位使用 `tier`，不使用 `cost`。
- 队伍状态以 `captains[].team` 为权威数据。
- 所有关键操作写入事件日志。
- AppState 必须可序列化，方便本地保存和后续迁移到服务端。

## 2. 基础类型

```ts
export type Tier = 1 | 2 | 3 | 4;

export type PlayerStatus = 'available' | 'drafted' | 'excluded';

export type DraftPhase =
  | 'idle'
  | 'setup'
  | 'hexcore_draw'
  | 'round_start'
  | 'round_prompt'
  | 'turn_start'
  | 'captain_action'
  | 'drawn'
  | 'picked'
  | 'turn_end'
  | 'round_end'
  | 'completed'
  | 'paused';
```

## 3. Player

```ts
export type Player = {
  id: string;
  name: string;
  gameId: string;
  score: number;
  tier: Tier;
  status: PlayerStatus;
  teamId?: string;
  tags?: string[];
  meta?: {
    s1?: number;
    s2?: number;
    s3?: number;
    note?: string;
  };
};
```

旧字段迁移：

```text
old.cost -> new.tier
old.manualCost -> new.manualTier
old.gold 相关字段 -> 丢弃
```

## 4. Captain

```ts
export type Captain = {
  id: string;
  name: string;
  userId?: string;
  playerId?: string;
  team: TeamMember[];
  status: 'active' | 'completed' | 'skipped';
};

export type TeamMember = {
  playerId: string;
  name: string;
  gameId: string;
  tier: Tier;
  source:
    | 'normal_pick'
    | 'blind_pick'
    | 'auto_assign'
    | 'hexcore'
    | 'compensation'
    | 'referee_override';
};
```

## 5. DraftSession

```ts
export type DraftSession = {
  id: string;
  phase: DraftPhase;
  round: number;
  maxRounds: 4;
  baseOrder: string[];
  currentOrder: string[];
  currentIndex: number;
  currentCaptainId?: string;
  currentDraw?: DrawState;
  runtimeEffects: RuntimeEffect[];
  temporaryTurns: TemporaryTurn[];
};
```

## 6. DrawState

```ts
export type DrawState = {
  id: string;
  captainId: string;
  round: number;
  effectiveTier: Tier;
  cards: DrawCard[];
  visibility: 'normal' | 'blind' | 'masked' | 'swapped_identity';
  seed?: string;
};

export type DrawCard = {
  slotId: string;
  playerId: string;
  visibleToReferee: boolean;
  visibleToCaptain: boolean;
  revealAfterPick?: boolean;
};
```

当前阶段由裁判代执行，裁判可以看到全部数据。但仍保留 `visibleToCaptain`，为未来队长端做准备。

## 7. HexcoreDefinition

```ts
export type HexcoreDefinition = {
  id: number;
  name: string;
  rarity: 'silver' | 'gold' | 'prismatic';
  trigger:
    | 'on_acquire'
    | 'round_start'
    | 'round_start_prompt'
    | 'before_turn'
    | 'before_draw'
    | 'after_draw'
    | 'before_pick'
    | 'after_pick'
    | 'manual';
  maxUses: number;
  maxUsesPerRound?: number;
  rounds?: number[];
  effects: HexcoreEffect[];
  tags?: string[];
};
```

## 8. RuntimeEffect

```ts
export type RuntimeEffect = {
  id: string;
  source: 'hexcore' | 'rule' | 'referee';
  sourceId: string;
  ownerCaptainId?: string;
  targetCaptainId?: string;
  type: string;
  priority: number;
  expiresAt:
    | { type: 'round_end'; round: number }
    | { type: 'turn_end'; captainId: string; round: number }
    | { type: 'draw_count'; remaining: number }
    | { type: 'session_end' };
  payload: Record<string, unknown>;
};
```

## 9. AppState

```ts
export type AppState = {
  version: number;
  mode: 'referee_single_client';
  auth: {
    role: 'referee' | null;
  };
  settings: {
    totalTeams: number;
    playersPerTeam: number;
    tierNames: Record<Tier, string>;
  };
  players: Player[];
  captains: Captain[];
  draft: DraftSession;
  hexcores: {
    pool: HexcoreDefinition[];
    assignments: HexcoreInstance[];
  };
  events: DomainEvent[];
};
```
