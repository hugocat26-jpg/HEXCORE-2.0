# 阵营锁定 10 队金币商店模式实施规范

本文档是后续 Codex 实施的唯一执行规范。实现时必须严格按本文执行，不得临时扩展规则、混用旧海克斯、引入第三阵营，或重构与本需求无关的代码。

如果实现过程中发现本文与现有代码存在冲突，必须先停止并说明冲突点，不得自行改写规则。

## 1. 实施目标

将当前项目改为 `10 队双阵营金币商店模式`。

最终规则如下：

- 总人数固定为 50 人。
- 阵营只有两种：本地人、外地人。
- 本地人固定 25 人，外地人固定 25 人。
- 队伍固定 10 队。
- 本地队长固定 5 人，外地队长固定 5 人。
- 每队固定 5 人，包括 1 名队长和 4 名队员。
- 抽卡轮数固定 4 轮。
- 本地队长只能购买、抽取、获得本地人队员。
- 外地队长只能购买、抽取、获得外地人队员。
- 队长仍显示在自己阵营和自己评分费用池里，但必须标记为队长锁定，不可被抽取。
- 保留金币商店随机感。
- 旧海克斯不与新海克斯混用，本模式使用本文定义的 10 款新海克斯。

## 2. 禁止事项

实现时禁止做以下事项：

- 不要新增第三阵营。
- 不要保留旧海克斯和新海克斯混用。
- 不要允许跨阵营转队。
- 不要允许跨阵营购买、随机、补位或海克斯获得选手。
- 不要把队长移出选手库费用池展示。
- 不要把队长放进单独的队长卡池。
- 不要改变主题系统。
- 不要引入后端、数据库或新的运行时服务。
- 不要重构与阵营锁定金币商店无关的模块。
- 不要改变页面主布局结构，只做必要的信息展示和交互补充。

## 3. 数据结构

### 3.1 选手阵营字段

所有选手必须增加字段：

```js
camp: 'local' | 'outsider'
```

显示文案：

```js
local = 本地人
outsider = 外地人
```

所有导入、导出、状态归一化、示例数据都必须支持该字段。

### 3.2 队长阵营

队长阵营必须从绑定的队长选手推导：

```js
captainCamp(captainId) = captainPlayer(captainId).camp
```

不要把 `captain.camp` 作为权威数据源。可以缓存，但所有规则判断必须以绑定选手的 `camp` 为准。

### 3.3 队长展示规则

队长仍然属于选手库的一员：

```text
展示池 = 本阵营全部 25 人，包含队长
可抽池 = 本阵营 + 非队长 + 未入队 + 未禁用
```

队长必须显示在自己阵营、自己评分对应的费用池中，并标记为：

```text
队长锁定
```

队长不可被商店、海克斯、补位、手动补录作为队员目标。

## 4. 人数与流程强校验

抽卡开始前必须强校验。

必须满足：

- 总人数正好 50。
- 本地人正好 25。
- 外地人正好 25。
- 队伍正好 10。
- 每队人数固定 5。
- 抽卡轮数固定 4。
- 本地队长正好 5。
- 外地队长正好 5。
- 每个阵营可抽队员正好 20。
- 所有选手必须有阵营。
- 队伍内所有队员必须与队长同阵营。

不满足时：

- 阻止进入抽卡流程。
- 在流程检查区显示具体原因。
- 不允许用隐式修复绕过校验。

## 5. 评分与费用池规则

每个阵营独立评分、独立分费。

本地人 25 人按评分从高到低排序，分为 5 档，每档 5 人。外地人同理。

费用池规则：

```text
每个阵营：
5费 5 人，评分最高
4费 5 人
3费 5 人
2费 5 人
1费 5 人，评分最低
```

队长参与评分和费用池展示，但不可被抽取。

如果评分相同，必须使用稳定排序规则。建议按导入顺序或选手 id 作为第二排序键，避免每次刷新费用池变化。

## 6. 选手库 UI

选手库必须改为双阵营卡池展示：

```text
本地人卡池 25/25
  全部 / 1费 / 2费 / 3费 / 4费 / 5费

外地人卡池 25/25
  全部 / 1费 / 2费 / 3费 / 4费 / 5费
```

每个选手卡必须显示：

- 姓名
- 游戏 ID
- 阵营
- 费用
- 评分
- 状态：可选、队长锁定、已入队、禁用

队长卡必须明显显示 `队长锁定`，但不能从对应费用池中消失。

## 7. 金币商店规则

商店生成必须只看当前队长阵营。

每次开店或刷新商店时，生成最多 5 张卡。当前队长同阵营可抽人数不足 5 时，只生成实际可抽数量。

### 7.1 空费用等比放大规则

每生成 1 张卡时，执行以下步骤：

```text
1. 读取当前轮次基础费用概率。
2. 统计当前队长同阵营各费用是否还有可抽选手。
3. 移除没有可抽选手的费用。
4. 剩余费用按原始概率等比例放大。
5. 按放大后的概率抽中一个费用。
6. 从该费用、同阵营、可抽选手中随机取 1 人。
7. 当前商店内不重复展示同一名选手。
8. 重复直到生成 5 张，或同阵营可抽选手不足。
```

示例：

第 3 轮原始概率：

```text
1费 20%
2费 30%
3费 35%
4费 13%
5费 2%
```

如果当前队长同阵营 1费池为空，则剩余概率变为：

```text
2费：30 / 80 = 37.5%
3费：35 / 80 = 43.75%
4费：13 / 80 = 16.25%
5费：2 / 80 = 2.5%
```

如果 1费和 2费都为空，则变为：

```text
3费：35 / 50 = 70%
4费：13 / 50 = 26%
5费：2 / 50 = 4%
```

这类概率变化是本模式允许的博弈结果。不要用低费优先补位规则替代。

### 7.2 购买硬校验

购买前必须二次校验：

```js
player.camp === captainCamp(captainId)
```

如果不一致：

- 拒绝购买。
- 拒绝入队。
- 写入事件日志。

该校验必须在 `assignment` 层存在，不能只依赖 UI 或商店过滤。

## 8. 最终补位规则

四轮结束后，如果队伍未满，只能从同阵营剩余可抽选手中随机补位。

补位候选必须满足：

```text
同阵营
非队长
未入队
未禁用
```

如果同阵营没有可补选手，则记录警告，不允许跨阵营补位。

## 9. 海克斯总原则

本模式不沿用旧海克斯池。旧海克斯与新海克斯不得混用。

所有新海克斯必须遵守：

```text
不能突破阵营限制。
不能让异阵营选手进入当前队伍。
不能选择队长作为队员目标。
```

如果海克斯目标涉及选手，默认只允许当前队长同阵营、非队长、可抽选手。

如果海克斯目标涉及队长，默认可选择同阵营队长。只有纯顺位类效果可允许跨阵营队长，但本文 10 款新海克斯中没有跨阵营队长效果。

## 11. 新海克斯执行规则

### 11.1 阵营侦察

使用时机：当前队长开店前。

效果：

- 下一次商店数量 +1。
- 额外卡必须来自当前队长同阵营可抽池。
- 仍只能购买 1 人。

失败条件：

- 当前队长已打开商店。
- 同阵营可抽人数不足以生成额外卡。
- 当前队伍已满。

### 11.2 定向招募

使用时机：当前队长开店前。

目标：选择一个位置，例如上路、打野、中路、下路、辅助。

效果：

- 下一次商店至少包含 1 名同阵营该位置可抽选手。
- 只影响本次商店。

失败条件：

- 当前队长已打开商店。
- 同阵营该位置没有可抽选手。
- 当前队伍已满。

### 11.3 压价券

使用时机：当前商店已打开，购买前。

效果：

- 本次购买费用 -1。
- 最低费用为 1 金币。

失败条件：

- 当前没有商店。
- 本轮已经购买。
- 当前队伍已满。

### 11.4 保留席位

使用时机：当前商店已打开，购买前。

目标：当前商店中的 1 张未购买卡。

效果：

- 如果本轮刷新商店，该卡保留在新商店里。
- 只保留 1 张。

失效条件：

- 购买成功。
- 跳过本轮。
- 进入下一位队长。

### 11.5 加急调货

使用时机：当前商店已打开，购买前。

目标：当前商店中的 1 张卡。

效果：

- 将该卡替换为同阵营、同费用、未出现在当前商店中的另一名可抽选手。
- 不消耗金币。
- 仍只能购买 1 人。

失败条件：

- 没有同阵营同费用替换目标。
- 当前没有商店。
- 当前队伍已满。

### 11.6 阵营封锁

使用时机：目标同阵营队长开店前。

目标：1 名同阵营、尚未行动的队长。

效果：

- 目标队长下一次商店卡位 -1。
- 最低商店数量为 3 张。
- 仅目标下一次开店生效。

失败条件：

- 目标已经行动。
- 目标不是同阵营。
- 目标队伍已满。

### 11.7 抬价干扰

使用时机：目标同阵营队长购买前。

目标：1 名同阵营队长。

效果：

- 目标本次购买费用 +1。
- 最高费用不超过 5 金币。
- 仅一次购买生效。

失败条件：

- 目标不是同阵营。
- 目标本轮已购买。
- 目标当前没有商店。

### 11.8 顺位插队

使用时机：自己行动前。

效果：

- 自己和本轮尚未行动的前一位队长交换。
- 不能越过已经行动过的队长。
- 每轮最多生效一次。

失败条件：

- 当前队长已经是可交换范围内第一位。
- 前一位队长已经行动。
- 当前轮已使用过该效果。

### 11.9 预算返还

触发时机：购买后自动触发。

效果：

- 如果本次购买的是 1费或 2费选手，返还 1 金币。
- 每队全局最多触发 1 次。

失败条件：

- 购买 3费及以上不触发。
- 已触发过不再触发。

### 11.10 稳健补强

使用时机：自己跳过购买前。

效果：

- 不打开或放弃当前商店。
- 系统从同阵营当前最低可用费用池随机分配 1 名选手。
- 消耗本轮购买权。

候选规则：

```text
从当前队长同阵营中查找最低费用的可抽池。
只允许非队长、未入队、未禁用选手。
```

失败条件：

- 同阵营没有任何可抽选手。
- 队伍已满。
