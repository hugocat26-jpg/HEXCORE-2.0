# 版本更新记录

本文件记录 HEXCORE 2.0 每次版本号变化的用户可见改动、规则变更和维护事项。每次提交并推送 GitHub 前，如果版本号发生变化，必须同步更新本文件。

## v2.0.22 - 2026-05-30

### 安装版修复

- Win11 启动脚本新增端口预检，若 `4186/4196` 已被旧容器或开发版占用，会先给出明确提示，避免直接显示 Docker 底层 bind error。
- 已验证当前安装目录在清理异常容器状态后可正常启动 Docker PostgreSQL 版本。

## v2.0.21 - 2026-05-30

### 安装版修复

- 修复 Win11 安装版 `Start-HEXCORE2.ps1` 在 Windows PowerShell 5.1 下因 UTF-8 无 BOM 中文字符串被误读导致的解析失败。
- 将 Windows 一键启动、停止、打开页面和日志脚本的运行时提示改为 ASCII 文案，避免客户电脑本地代码页影响脚本执行。
- 重新生成客户发布安装包，发布产物命名为 `HEXCORE2_Setup_v2.0.21.exe`。

## v2.0.20 - 2026-05-30

### 交付部署

- 完成 M13 Docker PostgreSQL 验证与一键部署交付：新增 Win11 启动、停止、打开页面和查看日志脚本，首次启动会从 `.env.example` 生成本机 `.env` 并随机生成 PostgreSQL 密码。
- 新增 Inno Setup 安装器工程，发布产物命名为 `HEXCORE2_Setup_v2.0.20.exe`；安装包不内置 Docker Desktop，只检测并引导安装。
- 新增 `deploy/baota/` 宝塔部署、更新、备份、状态脚本和 Nginx 反代模板，支持 `https://hexsss.com` 同源入口下的 Docker Compose + PostgreSQL 单机部署。
- 新增 `npm run verify:docker-postgres` 自动验收脚本，覆盖 Compose、PostgreSQL、三端加入、开店、购买、跳过、SSE 同步、容器重启恢复和备份/恢复。
- Win11 客户安装说明、多人端部署运维说明、腾讯云宝塔大陆部署说明和文档索引已同步更新。

### 体验与稳定性

- 创建赛事前的赛事 ID 校验改为读取房间列表，不再用预期 404 的 `/snapshot` 请求判断不存在，避免浏览器 console 出现正常流程里的 network error。
- 页面新增空 favicon 声明，避免浏览器默认请求 `/favicon.ico` 造成 404 日志。
- Docker PostgreSQL 环境已通过自动验收；本机 Edge 页面验收确认 `http://127.0.0.1:4186/` 页面非空、可创建 12 队无阵营房间、无错误覆盖层、Network 和 console 均无 error/warn。

### 安全边界

- `.env.example` 可提交，真实 `.env`、数据库密码、备份 dump、Docker volume 数据和本机临时验收产物不进入 Git。
- 宝塔备份/状态脚本不再 `source .env`，不通过 Docker CLI 参数传入 PostgreSQL 密码；备份在 `postgres` 容器内部读取容器环境变量执行。
- `/health` 只返回存储状态、房间数量、session TTL、SSE 订阅等运维信息，不泄漏连接串、密码、房间码、`sessionToken` 或 `streamToken`。

- 文档结构拆分为 `planning`、`product`、`architecture`、`multiplayer` 和 `operations`，当前计划、已完成记录、执行记录和候选计划分开维护。

### 安全边界

- 多房间请求继续以 `tournamentId + sessionToken` 双重绑定校验；房间列表和公开投影不返回房间码明文、房间码摘要或长期 `sessionToken`。
- 队长端与观众端投影继续脱敏裁判备注、替补缺席原因、档案风险说明和内部审计字段。
- Docker PostgreSQL 部署文档补齐本地正式部署、局域网访问、备份恢复和 active 房间上限配置。

## v2.0.16 - 2026-05-28

- JSON 文件和 SQLite 持久化继续只保存 session 摘要，不保存 `sessionToken` 明文。

## v2.0.15 - 2026-05-27

### 多人端

- M5 权威结算继续推进：服务端开始维护队伍金币、刷新次数、购买权和跳过状态。
- 队长刷新商店时由服务端按 `refreshCosts` 扣费并推进刷新次数，队长 payload 不能伪造刷新费用。
- 队长购买服务端商店卡时由服务端按卡位价格扣金币，并同步购买后的金币余额。
- 队长跳过本轮时，服务端会清空当前商店并推进到下一队。
- 当前轮最后一名队长跳过并绕回下一轮时，服务端会统一发放下一轮 `roundIncome`，并通过公开投影同步各队金币余额和本次轮次收入摘要。
- 购买或跳过后服务端会固化本轮权限，队长不能再通过重复开店或刷新商店重置购买状态。
- 雪定饿的喵进入服务端权威最小闭环：队长发动后由服务端登记目标下一次商店扰乱，目标开店时只打乱公开选手信息，不改变服务端真实槽位、价格或购买归属。
- 骤雨 血雾 清风进入服务端权威最小闭环：队长发动后服务端按当前队伍顺序登记最多 3 名目标的商店扰乱，目标公开商店隐藏身份但仍按真实槽位和价格结算。
- 海浪，我没吃饭进入服务端权威购买拦截最小闭环：已有本轮海浪监听时，服务端在购买后判定同阵营夺取或异阵营退回，统一返还原购买队长金币和购买权。
- 海浪持有者本轮跳过时，服务端会登记本轮海浪监听并清零该队金币，后续购买不再依赖裁判预置内部监听状态。
- 被海浪命中的原购买队长会获得 1 次服务端记录的免费刷新补偿；下一次刷新费用为 0，并在刷新后消费该补偿。
- 服务端目标型扰乱补上海浪免疫边界：雪定饿的喵和骤雨 血雾 清风不能直接指定海浪免疫队伍，血雾顺延链也会跳过海浪免疫队伍。
- 海浪异阵营命中后的轮末补偿进入服务端权威第一版：进入下一轮前，服务端会从海浪队伍同阵营可用选手池补入 1 名队员，并公开安全结算摘要。
- 海浪轮初自动触发进入服务端权威第一版：每轮服务端从持有海浪的队伍中稳定选出 1 名触发者；触发者尝试开店时会被服务端自动登记海浪并跳过，不生成或泄漏该队商店。
- 海浪购买命中概率改为服务端可重放 1/N 判定，公开摘要同步判定分母与 roll；队长端/观众端会保存并展示服务端 `lastHungryWave`，刷新或 SSE 重连后仍能看到最近海浪结果。
- 多人端服务新增本地文件持久化模式：设置 `HEXCORE_DATA_FILE` 后，赛事、房间凭据摘要和 session 会写入 JSON 文件；服务重启后可继续读取赛事并提交 command。
- 多人端服务新增本地 SQLite 数据库存储模式：设置 `HEXCORE_SQLITE_FILE` 后，赛事、房间凭据摘要和 session 会写入 SQLite 文件；服务重启后可继续读取赛事、管理房间并提交 command。
- 内置浏览器验证通道已恢复：多人房间加入页可在真实浏览器中打开并确认排版正常。
- 多人房间加入页易用性增强：创建赛事后可复制全部房间码、仅复制队长码、仅复制观众码或下载 TXT；加入页会记住最近服务地址，并按服务不可达、赛事不存在、加入码无效等情况给出更明确提示。
- 队长端和观众端顶栏新增身份状态条，直接显示“我的队伍/观众端、当前回合、当前权限”，降低非本人回合和只读模式下的误解。

### 安全边界

- 刷新金币不足时 command 被拒绝，状态版本不推进，金币不会被扣成负数。
- 公开投影只同步队伍金币余额、公开商店、购买状态和轮内状态，不公开完整私有选手池。
- 海浪内部监听状态、海克斯持有状态和私有选手池不进入公开快照；公开端只看到安全的最近海浪结算摘要。
- SQLite 存储仍不保存房间码明文或 `sessionToken` 明文；公开投影不输出内部 `snapshot.tournamentId`，仅保留顶层赛事 ID。

## v2.0.12 - 2026-05-27

### 多人端

## v2.0.10 - 2026-05-27

## v2.0.6 - 2026-05-27

### 多人端

- 服务端权威快照新增公开商店、轮内状态和海克斯响应窗口投影。
- `ShopOpened`、`ShopRefreshed`、`ShopCardPurchased`、`TurnSkipped` 和 `HexcoreUsed` 事件会更新可公开快照，供多端同步使用。
- 前端接收服务端快照或 SSE 事件时，会应用当前商店、轮内状态和海克斯窗口，减少队长端、观众端长期只依赖本地过渡状态的风险。

### 安全边界

- 公开投影会隐藏被打乱商店中的真实暗牌 ID，仅输出当前可见的选手标识。
- 队长端 command 不允许通过 payload 伪造权威商店卡或海克斯窗口；完整投影只接受裁判/管理员可信动作或后续服务端规则生成结果。

### 验证

- 回归新增服务端投影断言，覆盖商店、轮内状态、海克斯窗口、队长伪造投影被忽略和暗牌 ID 不泄漏。
