# 版本更新记录

本文件记录 HEXCORE 2.0 每次版本号变化的用户可见改动、规则变更和维护事项。每次提交并推送 GitHub 前，如果版本号发生变化，必须同步更新本文件。

## v2.0.6 - 2026-05-27

### 多人端

- 服务端权威快照新增公开商店、轮内状态和海克斯响应窗口投影。
- `ShopOpened`、`ShopRefreshed`、`ShopCardPurchased`、`TurnSkipped` 和 `HexcoreUsed` 事件会更新可公开快照，供多端同步使用。
- 前端接收服务端快照或 SSE 事件时，会应用当前商店、轮内状态和海克斯窗口，减少队长端、观众端长期只依赖本地过渡状态的风险。

### 安全边界

- 公开投影会隐藏被打乱商店中的真实暗牌 ID，仅输出当前可见的选手标识。
- 队长端 command 不允许通过 payload 伪造权威商店卡或海克斯窗口；完整投影只接受裁判/管理员可信动作或后续服务端规则生成结果。

### 验证

- 回归新增服务端投影断言，覆盖商店、轮内状态、海克斯窗口、队长伪造投影被忽略和暗牌 ID 不泄漏。

## v2.0.3 - 2026-05-26

### 新增赛制

- 赛程页新增“S7 班德尔保卫战”：两天 5x5 全交叉阵营积分赛，每天 25 场，两天共 50 场。
- 本地队伍作为班德尔阵营，外地队伍作为入侵者阵营；每场可录入比分和约德尔登场人数。
- 系统自动计算阵营积分：班德尔胜场 +1、入侵者胜场 +1、约德尔每登场 1 人班德尔 +0.5。
- 50 场完成后，分差大于 5 直接决出获胜阵营；分差不超过 5 自动开启“最强约德尔人 vs 最强侵略者”BO5，胜方额外 +10 分。
- 赛程页新增阵营积分榜、队伍贡献榜、Day 1 / Day 2 矩阵和阵营胜利展示。

### 验证

- `npm test` 通过，回归 48 项。
- `git diff --check` 通过，仅有 Git 行尾转换提示。
- 内置 Browser 验证班德尔保卫战生成后显示 2 个 Day 面板、50 张比赛卡、阵营积分和约德尔登场字段，控制台无 warning/error。
